# Architecture principles

## Document Control

- **Status:** Review
- **Owner:** Founder
- **Reviewer:** Repository Guardian
- **Approval Authority:** Founder
- **Version:** 0.1.0
- **Last Updated:** 2026-07-03
- **Last Review:** 2026-07-03
- **Lifecycle:** Review

- Inquiry-first commerce; catalog discovery leads to a qualified inquiry, not public checkout.
- No public price, cart, checkout, payment, or price-bearing structured data.
- Persian is the primary language; all presentation is RTL-aware.
- Mobile performance is the baseline, not a later optimization.
- SEO, accessibility, privacy, security, and observability are release criteria.
- Third-party and licensed products are reproducible through approved private delivery channels.
- Environment configuration and secrets are separated from source code.

## Governance Note

These existing principles require source, lifecycle, and approval confirmation before they can be treated as approved architecture authority.

The authoritative Core Project Principles are maintained in the [Project Bible](../00_PROJECT_BIBLE.md#core-project-principles). This supporting document cannot redefine them.
