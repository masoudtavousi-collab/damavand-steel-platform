## Metadata

- Use lifecycle values defined in [Document Lifecycle](12_DOCUMENT_LIFECYCLE.md).
- Use an ISO `YYYY-MM-DD` date for Last Updated.
- Increment versions according to the approved documentation versioning rule.
- TODO (Founder Decision Required): approve the documentation versioning rule.
- Apply the [Repository Metadata Standard](REPOSITORY_METADATA.md) to new documents and authorized header-normalization work.

## Headers

- Use one level-one heading for the title.
- Use level-two headings for primary sections.
- Do not skip heading levels.
- Keep headings unique within a document.

## Review Section

The Review section records assigned reviewers, review date, outcome, blocking findings, and approved exceptions. It does not replace detailed review evidence.
