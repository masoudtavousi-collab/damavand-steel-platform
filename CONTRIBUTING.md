# Contributing

Follow the proposed [Git Governance](docs/GIT_GOVERNANCE.md) for branch, commit, merge, version, tag, baseline, and history rules. Until Founder approval and the first approved baseline, it provides Review context only and does not authorize Git mutations.

Keep secrets and licensed archives outside Git. Update documentation and ADRs with material architectural or business-rule changes. All changes must preserve the Core Project Principles, including Persian RTL, Mobile First, Inquiry First, and No Public Pricing.
